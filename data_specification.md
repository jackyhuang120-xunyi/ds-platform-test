# 训练详情数据定义与指标计算规范

本文档详述了数字化运动康复平台上三类核心训练（等速、等张、等长）在详情页面的指标定义、数据来源及计算公式。

## 1. 全局共用信息
无论何种测试类型，以下信息均从数据库中直接读取：

| 模块 | 数据项 | 数据库来源字段 | 备注 |
| :--- | :--- | :--- | :--- |
| **基本信息** | 测试类别 | `type_name` (JOIN) | 映射为 type_id (1:等速, 2:等张, 3:等长) |
| | 测试部位 | `part_name` (JOIN) | 如“双腿”、“左臂”等 |
| | 测试组数 | `cfg_group` | 设备预设组数 |
| | 组间歇 | `cfg_rest_time` | 秒 |
| **个人信息** | 姓名/ID/组别 | `user_name`, `uid`, `group_name` | |
| | 用户体重 | `weight` | **核心字段**，用于所有相对指标计算 |
| | 活动范围 | `cfg_roma`, `cfg_romb` | 关节活动的起始/终止位置 (mm) |

## 2. 差异化指标与计算公式

### 2.1 等速测试 (Isokinetic)
等速模式下，表格分为“向心 (Concentric)”和“离心 (Eccentric)”两个维度，每个维度展现“最大值”与“平均值”。

#### 物理测量指标 (数据库直接映射)
*   **峰值力量 (kg)**：`con_stre_max`, `con_stre_avg`, `ecc_stre_max`, `ecc_stre_avg`
*   **峰值功率 (W)**：`con_power_max`, `con_power_avg`, `ecc_power_max`, `ecc_power_avg`
*   **做功 (J)**：`con_work_max`, `con_work_avg`, `ecc_work_max`, `ecc_work_avg`

#### 相对指标 (前端基于体重实时计算)
*   **相对峰值力量**：`物理力量 / 体重` (对应最大值/平均值)
*   **相对峰值功率 (W/kg)**：`物理功率 / 体重`
*   **相对做功 (J/kg)**：`物理做功 / 体重`

---

### 2.2 等张测试 (Isotonic)
等张模式侧重于在恒定阻力下的速度与功率表现。

*   **物理指标**：
    *   最大速度 (mm/s)：`con_speed_max`
    *   平均速度 (mm/s)：`con_speed_avg`
    *   最大功率 (W)：`con_power_max`
    *   平均功率 (W)：`con_power_avg`
*   **相对指标**：
    *   相对最大功率 (W/kg)：`con_power_max / weight`

---

### 2.3 等长测试 (Isometric)
等长模式侧重于固定角度下的最大静力表现。

*   **物理指标**：
    *   最大力量 (kg)：`con_stre_max`
    *   平均力量 (kg)：`con_stre_avg`
*   **相对指标**：
    *   相对最大力量：`con_stre_max / weight`
    *   相对平均力量：`con_stre_avg / weight`

---

## 3. 信号处理与可视化规范

### 3.1 过程曲线滤波 (Butterworth Filter)
为了消除 1000Hz 采样下的传感器噪声，所有曲线均经过处理：
*   **算法**：二阶巴特沃斯低通滤波器。
*   **参数**：截止频率 **15Hz**，采样率 1000Hz。
*   **相位处理**：采用**双向滤波 (Zero-phase)**，确保滤波后的峰值时刻无延迟偏移。

### 3.2 极性处理
*   **速度曲线取反**：在图表显示中，对 `speed` 数据进行取反处理 (`-speed`)。
*   **时间轴**：默认自适应加载全量采样点。

---

## 4. 数据异常处理
*   **空值显示**：统一使用 `--`。
*   **精度规范**：所有计算值和物理量统一保留 **2 位小数**。
